# 安全与受保护客户数据 Security

> 用到客户数据要按 **Protected Customer Data** 规则申请 + 落实数据保护;最小权限。
> 官方:https://shopify.dev/docs/apps/launch/protected-customer-data

---

## 访问级别

| Level | 范围 | 要求 |
|------|------|------|
| **0** | 不碰客户数据 | 无需操作 |
| **1** | 客户数据(不含姓名/地址/电话/邮箱) | 申请 + 满足 L1 |
| **2** | 含**受保护字段**(姓名 / 地址 / 邮箱 / 电话) | 申请 + 满足 L1&L2 + 参与数据保护审查 |

- 未批字段返回 `null`;未批类型返回 200 但 `errors` 里报错——代码要能处理。
- Partner Dashboard → App → **API access requests** 申请,只批**功能所需最小**。

## L1 要求（用到任何客户数据）
- 只处理**功能所需最小**个人数据;告知商家处理了什么、为什么。
- 尊重客户同意/退出;与商家签数据保护协议。
- 设**保留期**,**存储与传输都加密**;不留超过所需时长。

## L2 追加（用到姓名/地址/邮箱/电话）
- 备份加密;测试与生产数据分离。
- 有数据防泄漏策略 + 安全事件响应流程。
- 限制员工访问 + 强密码 + 访问日志。

## 权限 scopes
- **最小权限**:`shopify.app.toml` 只申请功能必需的 `access_scopes`(当前项目 `scopes=""`,加功能时按需补最小集)。

---

## ✅ Do / ❌ Don't
- ✅ 最小字段申请 + 加密存传 + 保留期;✅ 代码处理 `null`/redacted 与 `errors`。
- ❌ 申请用不到的字段/scope;❌ 明文存个人数据;❌ 无限期留存。

## 自检
- [ ] `access_scopes` 为最小必需集。
- [ ] 受保护字段按 Level 申请且理由充分。
- [ ] 个人数据加密 + 有保留/删除策略(配合 [webhooks-compliance.md](webhooks-compliance.md))。
