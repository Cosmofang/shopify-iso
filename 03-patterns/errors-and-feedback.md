# 反馈模式：错误 / 成功 / 加载

> 统一的反馈策略，覆盖 BFS 4.2.4（错误信息）与通用体验要求。

---

## 选对反馈组件

| 场景 | 用什么 | 为什么 |
|------|--------|--------|
| 字段级校验错误 | 字段 `error` 文案 | 就近、可行动（4.2.4） |
| 页面级错误/需阅读的信息 | Banner `tone="critical"` | 常驻、不打断 |
| 操作成功的瞬时反馈 | Toast | 轻量、自动消失 |
| 需确认的破坏性操作 | Modal（点击触发） | 阻断确认 |
| 引导/公告 | Banner（**不用自动 modal**） | 4.3.3 |

---

## 错误信息（4.2.4）

**红框 + 文案 + 指引三合一。**
```html
<s-text-field label="Domain" error="该域名已被占用，请换一个（只能含字母、数字、连字符）"></s-text-field>
```
文案模板：**「{哪里/什么}错了，{原因}，请{怎么做}」**。

## 加载状态

- 异步操作**必有** loading：按钮 `loading`、区块 skeleton、或 spinner。**别白屏**。
```html
<s-button variant="primary" loading>Saving…</s-button>
```

## 空状态

- 列表/表格为空时给**有意义的空状态** + 引导操作（不是空白）。

## 成功反馈
```js
shopify.toast.show('已保存');       // ✅ 成功
shopify.toast.show('保存失败', { isError: true }); // ✅ 失败=红
```

---

## ✅ Do / ❌ Don't
- ✅ 每个错误都可读、可行动。
- ✅ 每个异步都有 loading 与结果反馈。
- ❌ 只红框无字（4.2.4 打回）。
- ❌ 加载白屏、无进度感。
- ❌ 用自动弹窗做反馈（4.3.3）。

## BFS 注意
- **4.2.4**：字段错误必须有可行动文案。
- **4.3.3**：反馈不靠自动弹窗；红色只在错误。
- 通用：loading、空状态是审核常看项。
